prMetricPicker
directive in module pr.dashboard.widgets.util

Description

The prMetricPicker creates a picker for one or more metrics from a list of metric options and aggregation types.

It's model is the metrics attribute. metrics will be updated as the user adds them on the UI. metrics will update via two-way binding.

Attributes metric-options and aggregation-options added into the directive will display options to the user to create the list of metrics.

Usage

as element:
<pr-metric-picker
       metrics="{Array<object>}"
       multiple
       metric-options="{Array<object>}"
       aggregation-options="{Array<string>}">
</pr-metric-picker>

Directive info

  • This directive creates new scope.

Parameters

ParamTypeDetails
metricsArray<object>

The model, which contains the list of selected metrics. Each metric contains the following properties

  • name - {string=} - Column name in the data source query
  • type - {string=} - Aggregation type (count, sum, etc...)
  • alias - {string=} - Column alias in the data source query
multipleboolean

true if the user can input multiple metrics, false to allow one single metric.

metricOptionsArray<object>

The array of metric options Each metric option contains the following properties

  • name - {string=} - Name of the metric
aggregationOptionsArray<string>

Array of string containing the kinds of aggregations that are allowed for the metrics. (eg. "count", "sum", "min", "max", "unique")
