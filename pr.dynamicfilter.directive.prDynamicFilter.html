prDynamicFilter
directive in module pr.dynamicfilter

Description

Displays a list of filters for the user to drilldown in the data. The options for the user are loaded for the data source dynamically, using top N queries.

Optionally it emits ($emit) an event when filters are changed by the user so the data can be refreshed.

Usage

as element:
<pr-dynamic-filter
       datasource="{string}"
       table="{string}"
       [metric="{string}"]
       [filters="{object}"]
       [max-options="{number}"]
       [max-dimensions="{number}"]
       [model="{object}"]
       [drilldown="{boolean}"]
       [edit-mode="{boolean}"]
       [visible-options-selected="{number}"]
       [submit-event="{string}"]
       [dimensions="{array<object>}"]>
</pr-dynamic-filter>

Parameters

ParamTypeDetails
datasourcestring

The data source to be used

tablestring

The table in the data source to be queried

metric
(optional)
string

The metric to sort by in the top N query. Defaults to count.

filters
(optional)
object

Extra filters that can be added to the query.

maxOptions
(optional)
number

Limit to the number of options in each of the dimension filters. 4 - 10 is recommended.

maxDimensions
(optional)
number

Limit to the number of dimensions in each of the dimension filters. 4 - 10 is recommended.

model
(optional)
object

Key, value pairs of dimension names and arrays of values set by the filter, uses double data binding.

drilldown
(optional)
boolean

This flag makes the options in a filter depend on the filters previously selected. When a filter is selected, the filters below will be cleared of their values. Defaults to false

editMode
(optional)
boolean

This flag allows the user to add and remove dimensions in the filter on the UI.

visibleOptionsSelected
(optional)
number

Number of options that are shown before being collapsed as 2 of 4 options selected.

submitEvent
(optional)
string

Displays an extra submit button, which when click will $emit an event with the value of this parameter. The second argument is an object with the filters selected.

dimensions
(optional)
array<object>

Dimensions in the filters, uses double data binding. An additional option locked, which if true prevents the dimension from being removed.
